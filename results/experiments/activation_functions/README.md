# Activation Function Experiment

This experiment compares hidden-layer activation functions for the same
feed-forward neural network architecture.

The goal is to test whether changing the non-linearity improves the neural
network approximation of the Black-Scholes pricing function. The experiment
keeps the feature set fixed to the baseline inputs `(s0, k, t, r, sigma)` so
that the only changed parameter is the activation function.

## Configuration

All runs use the same random seed, dataset size, model widths, training
hyperparameters, and Monte Carlo benchmark settings.

ReLU command:

```bash
.venv/bin/python scripts/run_experiment.py \
  --n-samples 50000 \
  --max-epochs 100 \
  --batch-size 1024 \
  --mc-n-paths 20000 \
  --mc-evaluation-samples 256 \
  --feature-set base \
  --activation relu \
  --seed 42 \
  --data-dir data/experiments/base_relu \
  --output-dir outputs/experiments/base_relu
```

## Neural Network Metrics

| Experiment | MAE | RMSE | R2 | MAPE, price > 1 |
|---|---:|---:|---:|---:|
| `base + relu` | 0.1726914048 | 0.2297268905 | 0.9999036789 | 1.8686523438% |
| `base + tanh` | 0.1131617054 | 0.1654106055 | 0.9999500513 | 1.2877951860% |
| `base + silu` | 0.1077493951 | 0.1581683768 | 0.9999543428 | 1.2487852573% |

Relative error reductions versus ReLU:

| Experiment | MAE reduction | RMSE reduction | MAPE reduction |
|---|---:|---:|---:|
| `base + tanh` | 34.4717% | 27.9968% | 31.0843% |
| `base + silu` | 37.6058% | 31.1494% | 33.1719% |

## Interpretation

In this intermediate experiment, both Tanh and SiLU outperform ReLU. SiLU gives
the lowest MAE, RMSE, and MAPE among the three tested activations.

This result is plausible because Black-Scholes prices are generated by a smooth
analytical function. Smooth activations such as Tanh and SiLU may therefore
represent the target function more naturally than piecewise-linear ReLU under
this configuration.

The conclusion should remain cautious: this is an intermediate 50,000-sample
experiment. Before changing the final model choice in the report, the best
candidate activation should be confirmed with the final training configuration
and, ideally, together with the selected feature set.
